O Conceito de Fine-Tuning

  • Definio: o processo de utilizar um modelo pr-treinado como base e trein-lo adicionalmente em um dataset menor e especfico de um domnio ou tarefa.
  • Objetivo: Adaptar o modelo ao novo contexto, aprimorando o desempenho em aplicaes especializadas, como traduo de linguagem, anlise de sentimento ou sumarizao.
  • Vantagem: O fine-tuning se baseia no conhecimento pr-existente do modelo, o que reduz substancialmente os requisitos computacionais e de dados em comparao com o treinamento do modelo do zero (pre-training).

Categorias de Finetuning de LLMs. Fonte: Raschka.

Tipos de Fine-Tuning de LLMs

O fine-tuning pode ser categorizado pela natureza da superviso:

Tipo Caracterstica Aplicao
Supervisionado Utiliza dados rotulados adaptados tarefa alvo. Classificao de texto, tarefas de Gerao de Texto especficas.
No Supervisionado Expe o LLM a um grande corpus de texto no rotulado do domnio alvo. Refino da compreenso lingustica em novos domnios (ex: rea jurdica ou mdica).
Por Instruo Utiliza instrues em linguagem natural (Prompt Engineering) para treinar o modelo a seguir comandos. Criao de assistentes conversacionais especializados.

Pipeline Geral de Finetuning de LLMs

  • Estgio 1: Preparao do Conjunto de Dados
    • Limpeza, formatao e diviso do conjunto de dados para corresponder tarefa alvo.
  • Estgio 2: Inicializao do Modelo
    • Consiste em configurar os parmetros iniciais e as configuraes do LLM.
  • Estgio 3: Configurao do Ambiente de Treinamento
    • Configurao de hardware e software, hiperparmetros e arquitetura do modelo.
  • Estgio 4: Ajuste Fino Parcial ou Completo
    • o processo de atualizao dos parmetros do LLM utilizando o dataset especfico da tarefa.
 Seven Stage Fine-Tuning Pipeline for LLM. Fonte: Parthasarathy et al.

Pipeline Geral de Finetuning de LLMs (cont.)

  • Estgio 5: Avaliao e Validao
    • Mtricas como entropia cruzada e o monitoramento de curvas de perda (loss curves).
  • Estgio 6: Implantao
    • Execuo eficiente em plataformas de hardware ou software designadas e a implementao de medidas de segurana e sistemas de monitoramento.
  • Estgio 7: Monitoramento e Manuteno
    • Rastreamento contnuo do desempenho, atualizao do modelo conforme necessrio para se adaptar a novos dados ou requisitos em constante mudana.
 Seven Stage Fine-Tuning Pipeline for LLM. Fonte: Parthasarathy et al.

Desafios de Escalabilidade e Recursos

O fine-tuning de LLMs apresenta barreiras significativas:

  • Requisitos de Memria: O fine-tuning exige memria substancialmente maior do que a inferncia. Por exemplo, o LLaMA 2 (7B parmetros) em FP32 exige aproximadamente 112 GB de VRAM para o fine-tuning, limitando a acessibilidade para a maioria dos hardwares de consumo.
  • Otimizao de Recursos: A implantao eficiente requer tcnicas como Quantized LLMs (quantizao de modelos), que representam parmetros com menos bits (ex: 8-bit ou 4-bit) para reduzir o tamanho do modelo e otimizar a inferncia.

Consideraes ticas

O fine-tuning pode exacerbar questes ticas:

  • Vieses (Bias) e Justia (Fairness): Datasets de fine-tuning podem conter vieses que so transferidos e amplificados no modelo. crucial usar dados diversos e representativos.
  • Privacidade: O uso de dados sensveis exige tcnicas de preservao, como a Privacidade Diferencial e o Aprendizado Federado (Federated Learning).
  • Responsabilidade e Transparncia: Dada a capacidade do fine-tuning de alterar o comportamento do LLM, fundamental documentar detalhadamente o processo, o dataset e o impacto, utilizando frameworks como Model Cards ou AI FactSheets.

def download_and_unzip_spam_data(url, zip_path, extracted_path, data_file_path):
    if data_file_path.exists():
        print(f"{data_file_path} already exists. Skipping download and extraction.")
        return

    # Download
    with urllib.request.urlopen(url) as response:
        with open(zip_path, "wb") as out_file:
            out_file.write(response.read())

    # Unzip
    with zipfile.ZipFile(zip_path, "r") as zip_ref:
        zip_ref.extractall(extracted_path)

    # Extenso .tsv
    original_file_path = Path(extracted_path) / "SMSSpamCollection"
    os.rename(original_file_path, data_file_path)
    print(f"File downloaded and saved as {data_file_path}")

def create_balanced_dataset(df):
    num_spam = df[df["Label"] == "spam"].shape[0]

    # Seleciona amostras no-spam
    ham_subset = df[df["Label"] == "ham"].sample(num_spam, random_state=123)

    # Combina dados das duas classes
    balanced_df = pd.concat([ham_subset, df[df["Label"] == "spam"]])

    return balanced_df

def random_split(df, train_frac, validation_frac):
    # Embaralha o DataFrame
    df = df.sample(frac=1, random_state=123).reset_index(drop=True) # frac=1 means the entire dataset

    # Calcula ndices de treino e validao
    train_end = int(len(df) * train_frac)
    validation_end = train_end + int(len(df) * validation_frac)

    # Divide os dados
    train_df = df[:train_end]
    validation_df = df[train_end:validation_end] # to get an estimate during training how well the model performs
    test_df = df[validation_end:]

    return train_df, validation_df, test_df

def classify_review(text, model, tokenizer, device, max_length=None, pad_token_id=50256):
    model.eval()
    input_ids = tokenizer.encode(text)
    supported_context_length = model.pos_emb.weight.shape[1]

    # Truncamento
    input_ids = input_ids[:min(max_length, supported_context_length)]

    # Pad
    input_ids += [pad_token_id] * (max_length - len(input_ids))
    input_tensor = torch.tensor(input_ids, device=device).unsqueeze(0) # add batch dimension

    # Inferncia
    with torch.no_grad():
        logits = model(input_tensor)[:, -1, :]  # Logits of the last output token
    predicted_label = torch.argmax(logits, dim=-1).item()

    return "spam" if predicted_label == 1 else "not spam"
