1. Codificando uma Arquitetura de LLM

  • Modelos de linguagem de larga escala (LLMs), como o GPT, so arquiteturas de redes neurais profundas projetadas para gerar novo texto palavra (ou token) por palavra. Apesar do tamanho, a arquitetura do modelo menos complicada do que se imagina, j que muitos de seus componentes so repetidos.
  • A arquitetura de um GPT contm, ao lado das camadas de embedding, blocos transformer que incluem o mdulo de ateno multi-cabeas mascarada implementado anteriormente.

  • "n_heads" nmero de cabeas de ateno no mecanismo de multihead attention.
  • "n_layers" nmero de blocos transformer dentro do modelo, que iremos implementar em breve.
  • "drop_rate" descarta 10% das unidades ocultas durante o treinamento para mitigar overfitting.
  • "qkv_bias" decide se as camadas Linear no mecanismo de MHA devem incluir um vetor de bias ao calcular os tensores query (Q), key (K) e value (V); desativaremos essa opo, prtica padro em LLMs modernos.

import torch
import torch.nn as nn

class SimpleGPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
        self.pos_emb = nn.Embedding(
            cfg["context_length"], cfg["emb_dim"])
        self.drop_emb = nn.Dropout(cfg["drop_rate"])
        self.trf_blocks = nn.Sequential(
            *[SimpleTransformerBlock(cfg) for _ in range(cfg["n_layers"])])
        self.final_norm = SimpleLayerNorm(cfg["emb_dim"])
        self.out_head = nn.Linear(
            cfg["emb_dim"], cfg["vocab_size"], bias=False)
    
    def forward(self, in_idx):
        batch_size, seq_len = in_idx.shape
        tok_embeds = self.tok_emb(in_idx)
        pos_embeds = self.pos_emb(
            torch.arange(seq_len, device=in_idx.device))
        x = tok_embeds + pos_embeds
        x = self.drop_emb(x)
        x = self.trf_blocks(x)
        x = self.final_norm(x)
        logits = self.out_head(x)
        return logits

2. Layer Normalization

Usamos a normalizao de camada para melhorar a estabilidade e eficincia do treinamento de redes neurais.

  • Ideia central: ajustar as ativaes (sadas) de uma camada de rede neural de modo que tenham mdia zero e varincia unitria, tambm conhecida como unit variance.
  • Vantagem: Acelera a convergncia para pesos efetivos e garante um treinamento consistente e confivel.
  • Nas arquiteturas GPT2 e nos transformers, a normalizao de camada costuma ser aplicada antes e depois do mdulo de ateno multicabea e antes da camada de sada final.

class LayerNorm(nn.Module):
    def __init__(self, emb_dim):  # valores na dimenso do embedding
        super().__init__()
        self.eps = 1e-5
        # isso torna os valores treinveis
        self.scale = nn.Parameter(torch.ones(emb_dim))  
        self.shift = nn.Parameter(torch.zeros(emb_dim))

    def forward(self, x):
        mean = x.mean(dim=-1, keepdim=True)
        var = x.var(dim=-1, keepdim=True, unbiased=False)
        norm_x = (x - mean) / torch.sqrt(var + self.eps)  # eps evita diviso por zero
        return self.scale * norm_x + self.shift

        # shift no tem efeito aqui pois adiciona apenas zero; 
        # ser relevante mais tarde durante o treinamento,
        # permitindo que a rede desfaa essa normalizao; 
        # similarmente para scale

Feed-Forward com Ativao GELU

Vamos implementar uma camada composta de uma pequena rede neural que ser usada como parte do bloco Transfomer nos LLMs.

Iniciaremos com a funo de ativao.

Por que precisamos de funes de ativao no-lineares em Redes Neurais?

Importncia das funes de ativao no-lineares. Fonte: Anjali Kumari.

Feed-Forward com Ativao GELU

  • A funo de ativao a transformao no linear que fazemos ao longo do sinal de entrada.
  • Quando no temos a funo de ativao, os pesos e bias simplesmente fazem uma transformao linear. Uma equao linear simples de resolver, mas limitada na sua capacidade de resolver problemas complexos.
  • Introduzir nolinearidade nos neurnios, permitindo que redes profundas capturem padres complexos e faam classificaes em mltiplas classes.

GELU (Gaussian Error Linear Unit)

  • Presente nas arquiteturas GPT-2/3
  • Definio matemtica: $\text{GELU}(x) = x \cdot \Phi(x)$, onde $\Phi(x)$ a funo de distribuio acumulada da normal padro.
  • Propriedades:
    • Se comporta como identidade para valores positivos (mantm quase todos os sinais).
    • Para valores negativos, atenua suavemente o valor em vez de cortlo abruptamente como ReLU.
  • Aproximao prtica (usada na maioria das implementaes):
    $$ \text{GELU}(x) \approx 0.5\,x\,[1 + \tanh(\sqrt{\frac{2}{\pi}}(x + 0.044715x^{3}))] $$
  • Evita o clculo direto de funes exponenciais, reduzindo a carga computacional.
In[15]:
class GELU(nn.Module):
    def __init__(self):
        super().__init__()

    def forward(self, x):
        return 0.5 * x * (1 + torch.tanh(
            torch.sqrt(torch.tensor(2.0 / torch.pi)) * 
            (x + 0.044715 * torch.pow(x, 3))
        ))

class FeedForward(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]), GELU(),
            nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"]),
        )

    def forward(self, x):
        return self.layers(x)

A camada Feed-Forward crucial para melhorar a habilidade do modelo de generalizar os dados. Embora entrada e sada ambos tenham a mesma dimensionalidade, a camada escondida expande a dimensionalidade dos embeddings a um espao dimensional muito maior.

4. Conexes de Atalho (Residual Connections)

  • Propostas originalmente para mitigar os problemas do desaparecimento do gradiente.
  • Ideia: Adicionar a sada de uma camada sada de uma camada posterior, geralmente pulando (skipping) uma ou mais camadas no meio.
  • Uma conexo atalho cria um caminho alternativo e mais curto para o gradiente fluir atravs da rede.
In[19]:
class SimpleDeepNeuralNetwork(nn.Module):
    def __init__(self, layer_sizes, use_shortcut):
        super().__init__()
        self.use_shortcut = use_shortcut
        self.layers = nn.ModuleList([
            nn.Sequential(
                nn.Linear(layer_sizes[0], layer_sizes[1]), GELU()),
            nn.Sequential(
                nn.Linear(layer_sizes[1], layer_sizes[2]), GELU()),
            nn.Sequential(
                nn.Linear(layer_sizes[2], layer_sizes[3]), GELU()),
            nn.Sequential(
                nn.Linear(layer_sizes[3], layer_sizes[4]), GELU()),
            nn.Sequential(
                nn.Linear(layer_sizes[4], layer_sizes[5]), GELU())
        ])

    def forward(self, x):
        for layer in self.layers:
            layer_output = layer(x)
            # Verifica se o atalho pode ser aplicado
            if self.use_shortcut and x.shape == layer_output.shape:
                x = x + layer_output
            else:
                x = layer_output
        return x

class TransformerBlock(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.att = MultiHeadAttention(
            d_in=cfg["emb_dim"],
            d_out=cfg["emb_dim"],
            context_length=cfg["context_length"],
            num_heads=cfg["n_heads"], 
            dropout=cfg["drop_rate"],
            qkv_bias=cfg["qkv_bias"])
        self.ff = FeedForward(cfg)
        self.norm1 = LayerNorm(cfg["emb_dim"])
        self.norm2 = LayerNorm(cfg["emb_dim"])
        self.drop_shortcut = nn.Dropout(cfg["drop_rate"])

    def forward(self, x):
        # Shortcut connection for attention block
        shortcut = x
        x = self.norm1(x)
        x = self.att(x)  # Shape [batch_size, num_tokens, emb_size]
        x = self.drop_shortcut(x)
        x = x + shortcut  # Adiciona a entrada original de volta
        
        shortcut = x # Conexo de atalho
        x = self.norm2(x)
        x = self.ff(x)
        x = self.drop_shortcut(x)
        x = x + shortcut  # Adiciona a entrada original de volta

        return x

6. Modelo GPT

  • Vamos integrar o bloco Transformer na arquitetura GPT e montar uma verso totalmente funcional da verso original de 124 milhes de parmetros do GPT-2.
  • Tokenizao e Embeddings: O texto tokenizado convertido em embeddings de tokens, que so ento combinados com embeddings posicionais.
  • Empilhamento de Blocos Transformadores: O tensor resultante processado por uma srie de 12 blocos transformadores empilhados, cada um contendo camadas de ateno multi-cabea e redes neurais feed forward com dropout e normalizao de camada.
In[26]:
class GPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(
            cfg["vocab_size"], cfg["emb_dim"])
        self.pos_emb = nn.Embedding(
            cfg["context_length"], cfg["emb_dim"])
        self.drop_emb = nn.Dropout(cfg["drop_rate"])
        self.trf_blocks = nn.Sequential(
            *[TransformerBlock(cfg) for _ in range(cfg["n_layers"])])
        self.final_norm = LayerNorm(cfg["emb_dim"])
        self.out_head = nn.Linear(
            cfg["emb_dim"], cfg["vocab_size"], bias=False
        )

    def forward(self, in_idx):
        batch_size, seq_len = in_idx.shape
        tok_embeds = self.tok_emb(in_idx)
        pos_embeds = self.pos_emb(
            torch.arange(seq_len, device=in_idx.device))
        # Shape [batch_size, num_tokens, emb_size]
        x = tok_embeds + pos_embeds
        x = self.drop_emb(x)
        x = self.trf_blocks(x)
        x = self.final_norm(x)
        logits = self.out_head(x)
        return logits

7. Gerando Texto

  • Autoregessivo: Comeando com um contexto de entrada inicial, o modelo estima um token subsequente em cada iterao, anexando-o ao contexto de entrada para a prxima rodada de previso.
  • O processo pelo qual um modelo GPT vai dos tensores de sada para o texto gerado envolve vrias etapas que incluem decodificar os tensores de sada, selecionar tokens com base em uma distribuio de probabilidade e converter esses tokens em texto legvel por humanos.
In[32]:
def generate_text_simple(model, idx, max_new_tokens, context_size):
    # idx  um array (batch, n_tokens) de ndices no contexto atual
    for _ in range(max_new_tokens): # quantas tokens voc quer gerar
        # Trunca o contexto atual se ele exceder 
        # o tamanho de contexto suportado
        # Ex: se LLM suporta apenas 5 tokens, 
        #  e o tamanho do contexto  10 ento 
        #  apenas os ltimos 5 tokens so usados como contexto
        idx_cond = idx[:, -context_size:]
        
        with torch.no_grad():
            logits = model(idx_cond)
        
        # Foca apenas no ltimo passo de tempo
        # (batch, n_tokens, vocab_size) se torna (batch, vocab_size)
        logits = logits[:, -1, :] 

        # Aplica softmax para obter probabilidades
        probas = torch.softmax(logits, dim=-1)  # (batch, vocab_size)

        # Obtm o idx do vocabulrio com maior probabilidde
        idx_next = torch.argmax(probas, dim=-1, keepdim=True) 
        # argmax retorna a posio de ndice do valor mais alto

        # Anexa o ndice amostrado  sequncia em execuo
        idx = torch.cat((idx, idx_next), dim=1)  # (batch, n_tokens+1)

    return idx
